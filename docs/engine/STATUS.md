The engine is the mathematical and architectural foundation of the project. It includes the coordinate system (double-precision world coordinates with floating origin shifts for precision at vast distances), the simulation tick controller (a deterministic 10-step cycle that drives everything), the vessel container schema (the authoritative data shape for vessels), the three-mode physics architecture (PhysX-active for atmospheric and contact dynamics, Kepler-rails for orbital coast, interstellar-cruise for future inter-system travel), and the reference frame and patched conics work (body hierarchies, SOI re-rooting, multi-body orbital mechanics).

The engine is mostly complete. Phase 0 closed with the coordinate system, sim-tick controller, vessel container, two of three physics modes (PhysX-active and Kepler-rails), and the mode transition test infrastructure all in place. Phase 1 carryover work also completed: the Initialize-in-KeplerRails state-consistency fix and the per-sim-tick mode transition trigger evaluator infrastructure (disabled by default until upstream state systems exist). Most recently, SOI re-rooting landed with reference body hierarchies and patched conics for multi-body Phase 1.

Remaining engine work for Phase 1: the event prediction priority queue (the central new Phase 1 system, drives time-warp and analytic simulation of events like atmospheric entry, SOI crossings, scheduled burns), the time-warp rate machinery (depends on the event queue), and the save/load format (independent of other Phase 1 engine work). The Phase 1 validation milestone — placeholder cube launches, reaches orbit, transfers to a moon, captures into orbit, lands; time-warp works — gates Phase 2. The remaining engine work is on the critical path for that milestone.

The third physics mode (interstellar-cruise) is deferred to Phase 6 and is not on the v1 critical path.
